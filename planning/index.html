Development Plan
The Taste of Qin Yun Restaurant App

Team Members

Braden Chan, Harry Kwok, Hetarth Parikh, Leon Li, Uzair Sowdagar

Client

Feng

App Name

The Taste of Qin Yun

Purpose

Restaurant App that has online ordering.

Data Description

Table: MenuItems

Purpose: Stores all menu items available to customers

Data Stored: Item name, description, price, image, and availability

Fields:

  • item_id (INT, PRIMARY KEY)
  • name (VARCHAR)
  • description (TEXT)
  • price (DECIMAL)
  • image_url (VARCHAR)
  • availability (BOOLEAN)

Table: Orders

Purpose: Stores customer orders

Data Stored: Order details including items, quantities, and status

Fields:

  • order_id (INT, PRIMARY KEY)
  • customer_id (INT)
  • order_status (VARCHAR)
  • order_type (VARCHAR) (pickup/delivery)
  • total_price (DECIMAL)
  • created_at (DATETIME)

Table: OrderItems

Purpose: Links orders to menu items

Data Stored: Items included in each order and their quantity

Fields:

  • order_item_id (INT, PRIMARY KEY)
  • order_id (INT)
  • item_id (INT)
  • quantity (INT)

Table: Customers

Purpose: Stores customer information

Data Stored: Contact and login details

Fields:

  • customer_id (INT, PRIMARY KEY)
  • name (VARCHAR)
  • email (VARCHAR)
  • phone (VARCHAR)
  • password (VARCHAR)

App Functions

  • Browse menu → SELECT menu items from database
  • Filter menu → SELECT with conditions
  • Add to cart → client-side (JavaScript) + temporary storage
  • Place order → INSERT into Orders and OrderItems
  • Update order status (admin) → UPDATE Orders
  • Manage menu (admin) → INSERT, UPDATE, DELETE MenuItems
  • Login system → authentication using SELECT
